# Project overview

This project focuses on how content gets written, structured, validated, and published.

## Problem

Many personal sites become hard to maintain because page copy, metadata, and route structure are all managed ad hoc.

## Approach

I introduced a predictable schema for pages, projects, and blog posts so content can stay organized as the site grows.

## Key decisions

- Model content first, then build routes around it.
- Keep the launch workflow Git-based for simplicity.
- Validate front matter so content errors are caught early.

## Results

- Better content consistency
- Lower maintenance overhead
- Easier multilingual publishing
- Safer future expansion
